# Smart search — setup guide

This walks you through getting the search feature live on palestinelist.com.
It assumes zero prior experience with Cloudflare Workers.

There are three pieces:

1. **The search index** (`search-index.json`) — a JSON file listing every
   entry on the site. Generated by a Python script.
2. **The Cloudflare Worker** (`/worker/`) — a tiny proxy that holds your
   Thaura API key safely and talks to Thaura on the site's behalf.
3. **The browser code** (`/js/search.js`, `/js/lib/minisearch.min.js`,
   styles in `/css/styles.css`) — already in the repo, no setup needed.

## One-time: deploy the worker

You only do this once. Updates later are a single `wrangler deploy` command.

### 1. Install wrangler (Cloudflare's CLI)

In a terminal, run:

```
npm install -g wrangler
```

If you don't have `npm`, install Node.js first from https://nodejs.org/.

### 2. Log in

```
wrangler login
```

This opens a browser asking you to authorize. Use the same Cloudflare
account that already manages DNS for palestinelist.com.

### 3. Set your secrets

From the repo, change into the `worker/` folder:

```
cd worker
wrangler secret put THAURA_API_KEY
```

It will prompt for the value — paste your Thaura API key and hit Enter.

```
wrangler secret put ALLOWED_ORIGIN
```

When prompted, type: `https://palestinelist.com` and hit Enter.

(If you want to allow your local dev environment too, you can use a
comma-separated list, e.g.
`https://palestinelist.com,http://localhost:8000`.)

### 4. Deploy

```
wrangler deploy
```

When it finishes, it prints a URL like:

```
https://palestinelist-search.<your-subdomain>.workers.dev
```

**Copy that URL.** You need it for the next step.

### 5. Wire it into the site

Open `/js/search.js` and find this line near the top:

```js
const WORKER_BASE = 'https://palestinelist-search.YOUR-SUBDOMAIN.workers.dev';
```

Replace `YOUR-SUBDOMAIN` with whatever Cloudflare gave you in step 4.

(Optional but tidier: in the Cloudflare dashboard, go to your Worker →
Triggers → Add Custom Domain, and add something like
`search.palestinelist.com`. Then use that URL instead of the
`.workers.dev` one. It's just nicer to read in code.)

## Whenever you edit index.html

The search index is a static JSON file built from `index.html`. After you
add or change entries, regenerate it:

```
python3 tools/build-search-index.py
```

This writes `search-index.json` in the repo root. Commit and push as
usual. The script is idempotent — safe to re-run any time.

## Whenever you change the worker

If you edit `worker/worker.js` (to tweak prompts, rate limits, etc.):

```
cd worker
wrangler deploy
```

Your secrets stay set across deploys.

## What it costs

- **Cloudflare Workers**: free up to 100,000 requests/day. You will not hit
  this.
- **Thaura**: $0.50 per million input tokens, $2.00 per million output
  tokens. A typical "help me decide" call is about $0.0013 — under a
  fifth of a cent. Keyword search makes zero AI calls. Only the "Smart"
  toggle and "Help me decide" trigger Thaura.

## How to test it

After deploying:

1. Open palestinelist.com (or a local copy)
2. Click the magnifying-glass icon (or hit `⌘K` / `Ctrl+K`)
3. Type something like "Khalidi" or "Nakba" — you should get instant
   keyword results
4. Click the **Smart** pill, then type a natural-language query like
   "books written by Palestinian women" — Thaura should re-rank
5. Click **Help me decide what to read or watch**, fill out the guided
   form (or switch to "In your own words") and hit Recommend — Thaura
   returns three picks with short explanations

If something doesn't work, open the browser's DevTools → Console to see
the error.

## Troubleshooting

**"AI mode needs the worker"** in the footer when toggling Smart →
You haven't updated `WORKER_BASE` in `/js/search.js` yet (see step 5
above).

**`origin not allowed`** in the console →
The `ALLOWED_ORIGIN` secret doesn't match where you're testing from.
For local testing, add `http://localhost:8000` (or whatever port) to the
allowlist, then redeploy.

**`server is missing THAURA_API_KEY`** →
You skipped the secret-setting step. Run `wrangler secret put
THAURA_API_KEY` from the `worker/` folder again.

**`rate limited`** →
You hit the per-IP rate limit (30 requests/minute) — just wait a beat.
This protects your Thaura balance from runaway requests; raise it in
`worker.js` if you need to.

**Search results look stale after a deploy** →
Hard-refresh the browser (`Cmd+Shift+R` on Mac, `Ctrl+F5` on Windows).
The browser caches `search-index.json` and `search.js`.

## Files reference

| File | Purpose |
| --- | --- |
| `tools/build-search-index.py` | Python script that parses index.html into search-index.json |
| `search-index.json` | The catalog. Generated, committed. |
| `js/lib/minisearch.min.js` | Tiny fuzzy-search library, runs in the browser |
| `js/search.js` | Overlay, keyboard shortcuts, search logic, recommendation flow |
| `css/styles.css` | Styles for the search overlay (search section near the bottom) |
| `worker/worker.js` | Cloudflare Worker proxy — holds the Thaura key |
| `worker/wrangler.toml` | Worker config |
| `worker/README.md` | Worker-specific notes |
